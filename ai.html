AI 행동 규칙 설정

프로젝트별 CLAUDE.md로 AI의 동작 방식을 정밀하게 제어합니다.

CLAUDE.md
# SideProject — AI 운영 규칙
 
orchestrator:
  specialist_2_or_less: cto-lead
  specialist_3_plus: team-pool
 
model_assignment:
  cto_lead: "opus-4-6"
  security_architect: "opus-4-6"
  gap_detector: "opus-4-6"
  default: "sonnet-4-6"
 
quality_gate:
  match_rate_threshold: 90
  max_retry_loops: 3
 
hooks:
  - validate-commit
  - no-localstorage
  - task-qa-gate
01
오케스트레이터 자동 선택 specialist 2개 이하는 cto-lead 단독, 3개 이상은 팀 에이전트 풀 전체 활성화.
02
에이전트 모델 계층 고정 판단이 필요한 오케스트레이터만 opus, 실행 에이전트는 sonnet. 비용 최적화와 품질 균형.
03
PDCA 단계 게이트 강제 Phase 2.5 QA 시나리오 없이 구현 시작 금지. Plan 문서 없이 Design 진입 차단.
04
교차 검증 필수 구현한 에이전트가 자기 코드를 최종 승인하는 것 금지. 반드시 1차(code-reviewer) → 2차(cto-lead) 통과.
05
루프 차단 수정-재검증 최대 3회 초과 시 자동으로 사람에게 판단 요청. 무한 루프 방지.
06
코드 탐색 토큰 절약 Read/Grep 전에 qmd 인덱스(171개 파일) 우선 검색. 불필요한 컨텍스트 전송 차단.

Hooks (7)
⚡ PreToolUse — 실행 전 검증
validate-commit Bash
git commit 시 Conventional Commits 형식 자동 검증 — 형식 위반 시 커밋 차단
pre-push-qa Bash
git push 전 테스트 통과 강제 — 미검증 코드 원격 반영 차단
no-localstorage Write / Edit
민감 데이터의 localStorage 저장 시도 차단 — 보안 정책 자동 적용
subagent-verify Agent
등록되지 않은 에이전트 실행 경고 — CLAUDE.md 화이트리스트 기반 검증
📊 PostToolUse — 실행 후 추적
usage-tracker Agent
Agent 완료 시 opus/sonnet 호출 횟수 자동 기록 — 비용 모니터링
🛑 Stop — 세션 종료 전 게이트
task-qa-gate Stop
세션 종료 전 QA 미검증 완료 선언 경고 — 품질 게이트 강제
🚀 UserPromptSubmit — 메시지 전처리
auto-skill-trigger UserPromptSubmit
사용자 메시지 분석 후 관련 스킬 자동 감지 및 트리거 — 스킬 누락 방지

AI 비용 절감 전략

토큰 효율화와 모델 계층화로 비용 없이 품질을 유지합니다.

171
qmd 코드 인덱싱
171개 파일을 벡터 인덱스화. Read/Grep 대신 qmd 검색으로 컨텍스트 전송량 절감.
3:1
모델 계층 분리
판단 에이전트(opus) vs 실행 에이전트(sonnet) 분리. 오케스트레이터 3개만 opus 사용.
2
CLAUDE.md 계층화
SideProject 공통 규칙 + 프로젝트별 규칙 분리. 중복 컨텍스트 제거로 토큰 절약.
↓
.claudeignore 설정
node_modules, dist, .git 등 불필요 파일 컨텍스트 차단. 초기 로딩 토큰 최소화.
0
불필요 MCP 비활성화
Mintlify 등 미사용 MCP 서버 비활성화. 세션 시작 컨텍스트 크기 감소.
BG
Fire-and-Forget 패턴
비동기 작업은 응답 대기 없이 백그라운드 처리. 블로킹 없는 병렬 실행 구조.
